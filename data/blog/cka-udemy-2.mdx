# Manual Schedule

Pod를 기본 스케줄러 없이 수동으로 등록하는 방법이 있습니다. 기본적으로 스케줄러가 없다면 pod는 어떤 node에 배치될지 모릅니다. 따라서 pod 명세에 어떤 node에 등록될지 정해야합니다.

```yaml:pod.yaml {11} showLineNumbers
apiVersion: v1
kind: Pod
metadata:
  name: nginx
  labels:
    name: nginx
spec:
  containers:
  - name: nginx
    image: nginx
  nodeName: node02
```

## Label and Selector

시간이 지나면서 클러스터에 수백개 또는 수천개의 pod가 생겨날 수 있습니다. 수많은 pod를 구조적으로 운용하기 위해선 네이밍이 필요합니다.

pod에 레이블을 설정하고 셀렉터를 이용하여 pod 그룹을 선택합니다. 이를 통해 스케줄러 알고리즘을 설정할 수 있습니다.
